# 删除确认对话框修复 - 需求文档

## 1. 功能概述

修复删除操作的逻辑错误，将系统原生的 confirm 对话框替换为风格契合的自定义确认对话框。

## 2. 问题描述

当前系统中的删除操作存在以下问题：
- 使用原生的 `confirm()` 对话框，风格与应用不统一
- 多个组件中存在重复的确认逻辑
- 用户体验不够友好

## 3. 用户故事

### 3.1 作为用户，我希望删除操作有统一的确认对话框
**验收标准：**
- 所有删除操作使用统一的自定义确认对话框
- 对话框风格与应用整体设计保持一致
- 对话框支持自定义标题和消息内容

### 3.2 作为用户，我希望确认对话框操作清晰明确
**验收标准：**
- 对话框有明确的"确认"和"取消"按钮
- 按钮样式符合应用设计规范
- 支持键盘操作（Enter 确认，Escape 取消）

### 3.3 作为用户，我希望删除操作只在确认后执行
**验收标准：**
- 点击删除按钮后，先显示确认对话框
- 只有点击"确认"后才执行删除操作
- 点击"取消"或关闭对话框时，不执行删除操作

## 4. 功能需求

### 4.1 创建自定义确认对话框组件
- 组件名称：`ConfirmDialog.vue`
- 支持自定义标题、消息、确认按钮文本、取消按钮文本
- 支持通过 props 或组合式 API 调用
- 返回 Promise，根据用户选择 resolve(true) 或 resolve(false)

### 4.2 替换现有的 confirm() 调用
需要替换以下位置的 confirm() 调用：
- `src/components/VisualScriptEditor.vue` 中的 `deleteScript()`
- `src/components/VisualScriptEditor.vue` 中的 `deleteGroup()`
- `src/components/VisualScriptEditor.vue` 中的 `clearEvents()`
- `src/components/ScriptLibrary.vue` 中的 `deleteScript()`

### 4.3 任务删除操作添加确认
- `src/components/TaskDashboard.vue` 中的任务删除当前没有确认，需要添加确认对话框

## 5. 非功能需求

### 5.1 用户体验
- 对话框应有适当的动画效果（淡入淡出）
- 对话框应居中显示，有遮罩层
- 遮罩层点击应关闭对话框（等同于取消）

### 5.2 可维护性
- 确认对话框逻辑应集中管理，便于复用
- 代码应遵循 Vue 3 Composition API 最佳实践

### 5.3 兼容性
- 确保在 Tauri 应用环境中正常工作
- 支持键盘导航和操作

## 6. 技术约束

- 使用 Vue 3 Composition API
- 使用 TypeScript
- 样式使用 Tailwind CSS
- 保持与现有 ToastNotification 组件的设计风格一致

## 7. 验收测试场景

### 7.1 脚本删除确认
1. 在脚本编辑器中点击删除脚本按钮
2. 应显示自定义确认对话框
3. 点击"取消"，脚本不被删除
4. 再次点击删除，点击"确认"，脚本被删除

### 7.2 事件组删除确认
1. 在脚本编辑器中点击删除事件组按钮
2. 应显示自定义确认对话框
3. 点击"确认"，事件组被删除

### 7.3 清空所有节点确认
1. 在脚本编辑器中点击清空所有节点按钮
2. 应显示自定义确认对话框
3. 点击"确认"，所有节点被清空

### 7.4 任务删除确认
1. 在任务面板中点击删除任务按钮
2. 应显示自定义确认对话框
3. 点击"确认"，任务被删除

### 7.5 键盘操作
1. 打开确认对话框
2. 按 Escape 键，对话框关闭（等同于取消）
3. 再次打开对话框
4. 按 Enter 键，执行确认操作

## 8. 优先级

高优先级 - 影响用户体验和操作逻辑的正确性
