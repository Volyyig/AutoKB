# 深色模式手动测试指南

## 测试目标
验证需求 11.1：在浏览器中测试深色模式切换功能，验证所有组件在深色模式下的显示。

## 前置条件
- 开发服务器已启动 (pnpm dev)
- 浏览器已打开应用 (http://localhost:1420/)
- 浏览器开发者工具已打开 (F12)

## 测试步骤

### 第一部分：自动化控制台测试

1. **打开浏览器控制台**
   - 按 F12 打开开发者工具

2. **运行自动化测试脚本**

   - 复制整个脚本内容

   - 按 Enter 执行

3. **查看测试结果**

## 附录：常见问题排查

### 问题 1：切换主题后某些元素颜色不变
**可能原因：**
- 元素使用了硬编码颜色而非 CSS 变量
- CSS 变量未正确定义

**排查方法：**
1. 在开发者工具中选择该元素
2. 查看 Computed 样式
3. 检查颜色值是否来自 CSS 变量

### 问题 2：深色模式下文字不可读
**可能原因：**
- 文字颜色和背景色对比度不足
- 使用了错误的 CSS 变量

**排查方法：**
1. 检查 `--text-main` 和 `--background` 的值
2. 确保对比度足够（至少 4.5:1）

### 问题 3：主题不持久化
**可能原因：**
- localStorage 未正确保存
- applyTheme 函数未正确调用

**排查方法：**
1. 在控制台运行：`localStorage.getItem('theme-dark')`
2. 检查返回值是否正确
3. 检查 scriptStore.ts 中的 applyTheme 函数

### 问题 4：切换主题时有闪烁
**可能原因：**
- CSS 变量更新和 DOM 更新不同步
- 缺少过渡动画

**排查方法：**
1. 检查是否有 transition 属性
2. 考虑添加 `transition: background-color 150ms, color 150ms`
